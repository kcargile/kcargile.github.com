I've received a number of emails in response to my Creating a Singleton Configuration Manager article that appeared on the ASP.NET website a few weeks ago, many asking about the Singleton pattern and how best to implement it in .NET. Since this has been a pretty common question, I thought I'd share the answer that I've been giving with everyone:

First, if you're new to design patterns, there are several good books out there to read. The most ubiquitous (and the one that I consistently recommend), however, is Gamma et al's Design Patterns. One reader also recommended an excellent article on MSDN that lays out the Singleton pattern in .NET, complete with code samples and explanation of associated multithreading issues.

These two resources should give you all you need to get a Singleton class up and running correctly in .NET.
