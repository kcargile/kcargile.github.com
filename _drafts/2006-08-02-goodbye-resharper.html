Doug Rohm has given me a lot of grief over the last year or so about not giving his favorite Visual Studio plugin, CodeRush, a fair chance. Instead, despite its painfully slow startup times and sometimes quirky behavior, I've been using JetBrain's Resharper and have been fairly happy with it. Besides, I could never get CodeRush to install correctly on my laptop—probably not the fault of CodeRush itself but frustrating nonetheless.

The past few months, however, have stated to reshape my opinion of Resharper. Working with SQL scripts in Visual Studio seems to confuse Resharper, eventually causing it to throw an exception and usually requiring a restart. Likewise for complex, supposedly "invalid" ASPX pages—usually those containing inline code—often cause Resharper to throw exceptions with such wild abandon that Visual Studio eventually crashes (hope you saved recently.)

But because its pretty uncommon to run across these types of files, I figured I'd just disable Resharper while I worked with them. I figured wrong. You can't really turn it off. Almost any hotkey combination—say CTRL + SPACE—causes Resharper to reactivate itself, painful startup time included. Unacceptable. If I wanted software to take over my computer I'd just install AOL.

So, I've decided to retire Resharper. I guess I'll just have to get used to working without it. Maybe I'll see if I can finally get CodeRush to install. Happy, Doug?
